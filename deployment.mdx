---
title: 'Deploy Incidencias Backend to a production server'
description: 'Configure PHP, database, mail transport, public storage, and Firebase credentials for a production Laravel deployment of Incidencias Backend.'
---

Running Incidencias Backend in production requires a hardened Laravel environment with a real database, a working mail transport for PIN-based password recovery, a public storage disk for uploaded photos, and a Firebase service account for push notifications. This page covers each area and finishes with an optimized deployment checklist.

## Requirements

| Dependency | Minimum version | Notes |
|---|---|---|
| PHP | 8.2+ | Required by `composer.json` |
| MySQL | 8.0+ | SQLite is supported but not recommended for production |
| Composer | 2.x | Dependency manager |
| Node.js | 18+ | Required to build front-end assets with `npm run build` |

## Environment configuration

Start from `.env.example` and set the following values for a production deployment.

### Application

```bash
APP_ENV=production
APP_DEBUG=false
APP_URL=https://your-domain.com
```

<Warning>
  Never set `APP_DEBUG=true` in production. It exposes stack traces, environment variables, and other sensitive information in HTTP error responses.
</Warning>

### Database

```bash
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=incidencias
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password
```

### Mail

The application sends 6-digit PIN codes for password recovery. Configure a real SMTP transport:

```bash
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailprovider.com
MAIL_PORT=587
MAIL_USERNAME=your_smtp_username
MAIL_PASSWORD=your_smtp_password
MAIL_FROM_ADDRESS="noreply@your-domain.com"
MAIL_FROM_NAME="${APP_NAME}"
```

<Tip>
  If you use a transactional email service such as Mailgun, Postmark, or Amazon SES, set `MAIL_MAILER` to the corresponding Laravel driver (`mailgun`, `postmark`, or `ses`) and provide the driver-specific credentials.
</Tip>

### Storage

INE verification photos and incident images are stored on the `public` disk. Set `FILESYSTEM_DISK=public` in `.env`, then create the symbolic link so files are served over HTTP:

```bash
php artisan storage:link
```

This creates `public/storage` → `storage/app/public`. Without it, uploaded images will not be accessible from the web.

### Firebase credentials

Push notifications are sent through Firebase Cloud Messaging. Place your service account key file at:

```
storage/app/firebase-credentials.json
```

Download this file from the Firebase console under **Project settings → Service accounts → Generate new private key**. The file must be present before any notification dispatches are attempted.

<Warning>
  Do not commit `firebase-credentials.json` to version control. Add it to `.gitignore` and provision it separately on each server.
</Warning>

## Production deployment checklist

Run these commands in order after every deployment to production.

<Steps>
  <Step title="Install dependencies without dev packages">
    ```bash
    composer install --optimize-autoloader --no-dev
    ```
  </Step>

  <Step title="Cache the configuration">
    ```bash
    php artisan config:cache
    ```

    Merges all configuration files into a single cached file for faster boot times.
  </Step>

  <Step title="Cache the route list">
    ```bash
    php artisan route:cache
    ```
  </Step>

  <Step title="Run migrations">
    ```bash
    php artisan migrate --force
    ```

    The `--force` flag is required in production because Laravel prompts for confirmation in non-local environments.
  </Step>
</Steps>

## Admin access

The web panel at `/dashboard` is restricted to users whose `role` column is set to `'admin'` in the `users` table. There is no self-service way to promote an account — you must update the record directly in the database:

```sql
UPDATE users SET role = 'admin' WHERE email = 'admin@your-domain.com';
```

<Note>
  Admin users still need a valid session through Laravel Breeze's standard authentication routes. The `role` check is enforced inside the dashboard controller after login.
</Note>
