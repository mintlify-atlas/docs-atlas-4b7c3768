This quickstart covers everything you need to run Incidencias Backend on your local machine, register a citizen account with identity verification, and make your first authenticated API call. By the end you will have a working Laravel development server and a Sanctum access token ready for testing incident reports.

## Set up the project

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Dannygs17/incidencias-backend.git && cd incidencias-backend
    ```
  </Step>

  <Step title="Install PHP dependencies">
    ```bash
    composer install
    ```
  </Step>

  <Step title="Copy the environment file and generate an application key">
    ```bash
    cp .env.example .env && php artisan key:generate
    ```

    Open `.env` and update `DB_DATABASE` (and any other database credentials) before continuing.
  </Step>

  <Step title="Run database migrations">
    ```bash
    php artisan migrate
    ```
  </Step>

  <Step title="Link the public storage disk">
    ```bash
    php artisan storage:link
    ```

    This makes uploaded files (INE photos, incident images) accessible at `/storage/…` URLs.
  </Step>

  <Step title="Start the development server">
    ```bash
    php artisan serve
    ```

    The API is now available at `http://localhost:8000`.
  </Step>
</Steps>

## Register a citizen

New accounts require a CURP and two photos of the citizen's INE (front and back). Send the request as `multipart/form-data`:

```bash
curl -X POST http://localhost:8000/api/register \
  -F "name=Jane Doe" \
  -F "email=jane@example.com" \
  -F "password=secret123" \
  -F "curp=DOEJ900101MDFXXX01" \
  -F "ine_frente=@/path/to/ine_front.jpg" \
  -F "ine_reverso=@/path/to/ine_back.jpg"
```

A successful response returns HTTP `201` with an `access_token` and the newly created user object:

```json
{
  "message": "Registro exitoso.",
  "access_token": "1|xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "user": {
    "id": 1,
    "name": "Jane Doe",
    "email": "jane@example.com",
    "status": "pending"
  },
  "status": "pending"
}
```

<Note>
  Every new account starts with a `status` of `pending`. An administrator must approve the account through the web panel before the citizen can submit incident reports. Until then, protected endpoints will refuse requests from unapproved accounts.
</Note>

## Log in and get an access token

Once your account is approved, exchange credentials for a Laravel Sanctum access token:

```bash
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"email": "jane@example.com", "password": "secret123"}'
```

The response includes the `access_token` you will attach as a `Bearer` token to every subsequent authenticated request:

```json
{
  "access_token": "1|xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "user": {
    "id": 1,
    "name": "Jane Doe",
    "status": "approved"
  }
}
```

Pass the token in the `Authorization` header for all protected routes:

```bash
curl http://localhost:8000/api/user \
  -H "Authorization: Bearer 1|xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

For a full breakdown of the authentication flow — including Google sign-in, account verification, and password recovery — see the [authentication overview](/auth/overview).
