---
title: 'Recover your account with a 6-digit PIN code'
description: 'Request a one-time PIN to your registered email address, then submit it with your new password to complete the two-step account recovery flow.'
---

When a citizen forgets their password, they can regain access through a two-step PIN-based reset flow. First, they request a 6-digit PIN to be sent to their registered email address. Then they submit the PIN alongside their new password to complete the reset. The PIN is stored temporarily in the database and is invalidated immediately after a successful reset.

## Reset flow

<Steps>
  <Step title="Request a PIN">
    Send the citizen's email address to `/api/forgot-password`. The backend generates a random 6-digit PIN, saves it to the user record, and sends it via email.

    ```http
    POST /api/forgot-password
    Content-Type: application/json
    ```

    <ParamField body="email" type="string" required>
      Email address associated with the citizen's account.
    </ParamField>

    ```bash
    curl -X POST https://your-domain.com/api/forgot-password \
      -H "Content-Type: application/json" \
      -d '{"email": "citizen@example.com"}'
    ```

    The endpoint always returns `200` with the same message, regardless of whether the email exists in the database:

    ```json
    {
      "message": "Si el correo existe, enviaremos un PIN."
    }
    ```

    <Tip>
      The response is intentionally ambiguous. Returning an identical message whether or not the email exists prevents attackers from enumerating registered accounts.
    </Tip>
  </Step>

  <Step title="Reset the password">
    Submit the PIN from the email together with the new password to `/api/reset-password-pin`.

    ```http
    POST /api/reset-password-pin
    Content-Type: application/json
    ```

    <ParamField body="email" type="string" required>
      Email address used in step 1.
    </ParamField>

    <ParamField body="pin" type="string" required>
      6-digit PIN received by email.
    </ParamField>

    <ParamField body="password" type="string" required>
      New password. Minimum 6 characters.
    </ParamField>

    <ParamField body="password_confirmation" type="string" required>
      Must match the `password` field exactly.
    </ParamField>

    ```bash
    curl -X POST https://your-domain.com/api/reset-password-pin \
      -H "Content-Type: application/json" \
      -d '{
        "email": "citizen@example.com",
        "pin": "482951",
        "password": "newSecurePass",
        "password_confirmation": "newSecurePass"
      }'
    ```

    **On success** — HTTP `200`:

    ```json
    {
      "message": "Contraseña actualizada correctamente."
    }
    ```

    **On failure** — HTTP `400` (incorrect or expired PIN):

    ```json
    {
      "message": "El código PIN es incorrecto o ha expirado."
    }
    ```
  </Step>
</Steps>

<Note>
  The PIN is single-use. As soon as a successful reset is completed, the PIN is cleared from the database (`reset_pin` is set to `null`). Attempting to reuse the same PIN will return a `400` error.
</Note>
